# Project documentation

Index of the harness documentation (`harness/docs/`). Change it only when adding/removing files.

## Harness operation

| Doc                                | When to read                                                                                                                  |
| ---------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| `workflow.md`                      | Operational manual: session rituals, orchestrator–workers loop, hard rules, verification.                                     |
| `state/docs-update-guide.md`       | Before modifying any doc.                                                                                                     |
| `roadmap/dashboard.html`           | Human entry point for the roadmap (`pnpm roadmap`).                                                                           |
| `roadmap/mvp-example.json`         | Versioned roadmap template. Copy it to `mvp-*.json` (local, gitignored) for your work. Validate with `pnpm roadmap:validate`. |
| `roadmap/plans/example-tagging.md` | Versioned example of the required plan structure (context/data/flow/edge cases/scope/verification).                           |

## Product foundations
